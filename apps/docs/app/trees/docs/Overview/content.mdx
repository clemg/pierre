## Overview

<Notice variant="warning" icon={<IconCiWarningFill />}>
  File tree is in early active development—APIs are subject to change.
</Notice>

**File tree** is a library for rendering file trees on the web. It includes
vanilla JavaScript and React integration, with support for SSR. You get flexible
components for building trees with mutations, filtering, directory flattening,
icons, drag and drop, and customizable styling.

<FileTree
  options={overviewFileTreeOptions}
  className="overflow-hidden rounded-lg border-1 contain-layout contain-paint p-3 bg-neutral-50 dark:bg-neutral-900 max-w-[400px]"
  style={{
    '--trees-search-bg-override': 'light-dark(#fff, oklch(14.5% 0 0)',
  }}
/>

The recommended API is **`@pierre/trees/path-store`**, which provides an
imperative, path-first file tree built on top of `@pierre/path-store`. It
supports native mutations (`add`, `remove`, `move`, `batch`), always-on
virtualization, drag and drop, inline renaming, search, and composition slots
(headers, context menus). The UI renders in Shadow DOM for style isolation.

## Rendering Trees

Import `PathStoreFileTree` from `@pierre/trees/path-store`. Pass a list of file
paths and optional configuration; the tree builds the hierarchy and handles
expand/collapse, selection, search, and mutations.

### Legacy API [toc-ignore]

The legacy `FileTree` class (`@pierre/trees`) and React component
(`@pierre/trees/react`) are still available. See the
[Legacy Vanilla API](#vanilla-js-api) and [Legacy React API](#react-api)
sections for those APIs. New projects should use `@pierre/trees/path-store`.
