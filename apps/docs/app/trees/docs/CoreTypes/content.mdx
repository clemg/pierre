## Core Types

Before diving into the options and components, it's helpful to understand the
core data structures used throughout the path-store API.

### PathStoreFileTreeOptions

`PathStoreFileTreeOptions` is the main options object for `PathStoreFileTree`.
It defines the file list, instance id, flattening behavior, search mode, drag
and drop, composition slots, and other structural behavior.

#### The `paths` option [toc-ignore]

The tree is built from an array of file paths (strings). Folders are inferred
from path segments; use forward slashes. Top-level paths (e.g. `README.md`) and
nested paths (e.g. `src/components/Button.tsx`) are both valid. A trailing slash
(e.g. `src/components/`) marks an explicit directory, including empty folders.

<DocsCodeExample {...pathsOptionExample} />

That produces a tree with `README.md` and `package.json` at the root and a `src`
folder containing `index.ts`, `utils/helpers.ts`, and a `components` folder with
`Button.tsx`.

#### Options reference [toc-ignore]

`paths` is the only required option, all others are optional.

| Option                    | Description                                                                                                                                                                                        |
| ------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `paths`                   | **Required.** Array of file paths (strings). Defines the list and hierarchy.                                                                                                                       |
| `id`                      | Unique identifier for this instance (DOM ids, SSR). Default: auto-generated.                                                                                                                       |
| `flattenEmptyDirectories` | When `true`, single-child folder chains become one row (e.g. `build / assets / images` → `build · assets · images`). Default: `false`. [Live demo](/preview/trees#flattening).                     |
| `fileTreeSearchMode`      | Search behavior: `'expand-matches'` (default), `'collapse-non-matches'`, or `'hide-non-matches'`.                                                                                                  |
| `search`                  | Render the built-in search input and enable keyboard-open search behavior. Default: `false`.                                                                                                       |
| `dragAndDrop`             | Enable built-in drag and drop. `true` for defaults, or a `PathStoreTreesDragAndDropConfig` object. Default: `false`.                                                                               |
| `renaming`                | Enable inline renaming. `true` for defaults, or a `PathStoreTreesRenamingConfig` object. Default: `false`.                                                                                         |
| `gitStatus`               | Optional `GitStatusEntry[]` for Git-style file status (`added`, `modified`, `deleted`). Folders with changed descendants also receive a change indicator. [Live demo](/preview/trees#path-colors). |
| `icons`                   | Optional built-in icon set or `FileTreeIconConfig` for semantic colors and custom sprite overrides. [Live demo](/preview/trees#custom-icons).                                                      |
| `composition`             | Optional composition slots for header and context menu. See [Composition](#composition).                                                                                                           |
| `viewportHeight`          | Height of the virtualized viewport in pixels. Default: `320`.                                                                                                                                      |
| `itemHeight`              | Height of each tree row in pixels. Default: `28`.                                                                                                                                                  |
| `initialExpandedPaths`    | Array of directory paths to expand on mount (e.g. `['src/', 'src/components/']`).                                                                                                                  |
| `preparedInput`           | Pre-sorted input from `PathStore.preparePresortedInput` for faster initialization with large trees.                                                                                                |
| `onSelectionChange`       | Callback fired when the selection changes. Receives the array of selected paths.                                                                                                                   |
| `onSearchChange`          | Callback fired when the search value changes. Receives the search string or `null` when closed.                                                                                                    |

### PathStoreTreesSearchMode

`PathStoreTreesSearchMode` controls how search affects the tree. Pass it via
`fileTreeSearchMode` in options.

- `'expand-matches'` (default): expand nodes that match the search.
- `'collapse-non-matches'`: hide non-matching branches so only matching paths
  and their parents stay visible.
- `'hide-non-matches'`: keep branch structure but hide non-matching rows.

### Item Handles

Call `getItem(path)` on a `PathStoreFileTree` instance to get a typed handle for
interacting with a specific tree item. Handles provide methods for focus,
selection, and (for directories) expansion.
