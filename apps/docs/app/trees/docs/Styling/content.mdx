## Styling

The file tree is rendered inside a shadow DOM host (the `file-tree-container`
custom element), so its styles are isolated from your page.

Customize it by styling the host element: use `className` and `style` on the
React `FileTree` component, or style the host returned by
`getFileTreeContainer()` after `render()`. That lets you control layout
(`width`, `maxHeight`, borders, overflow) and override the tree's CSS variables
from global CSS, a wrapper, or the host element directly.

<Notice icon={<IconBulbFill />}>
  Think of styling in two layers: host styles change the outer panel, and
  `--trees-*` variables change the UI rendered inside the shadow DOM.
</Notice>

### Unsafe CSS escape hatch (legacy only)

The `unsafeCSS` option is available on the legacy `FileTree` class to inject raw
CSS into the tree shadow root. This option is **not available** on the
path-store `PathStoreFileTree`. Use `--trees-*` CSS variables for customization
instead.

### Fallback order

Trees uses a consistent fallback chain for colors and theme tokens:

1. `--trees-*-override`
2. `--trees-theme-*`
3. library defaults

That means explicit overrides always win. Theme-derived values from
`themeToTreeStyles()` sit in the middle, and the built-in defaults are only used
when neither of those is present.
