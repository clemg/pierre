## SSR

<Notice icon={<IconBulbFill />}>
  Import `preloadPathStoreFileTree` from `@pierre/trees/path-store`.
</Notice>

The SSR API lets you pre-render the file tree HTML on the server for fast first
paint, then hydrate on the client for full interactivity.

### Usage

`preloadPathStoreFileTree` returns a payload object: `{ id, shadowHtml, html }`.
Render `payload.html` into the page on the server (e.g. via
`dangerouslySetInnerHTML`), then create a `PathStoreFileTree` with the same
options on the client and call `hydrate({ fileTreeContainer })`.

<Notice variant="warning" icon={<IconCiWarningFill />}>
  Options used for pre-rendering must exactly match what you pass on the client.
  Use the same `PathStoreFileTreeOptions` object (or equivalent) when calling
  `preloadPathStoreFileTree` and when constructing `PathStoreFileTree` so the
  tree structure and state hydrate correctly.
</Notice>

### Legacy SSR [toc-ignore]

The legacy `preloadFileTree` from `@pierre/trees/ssr` is still available for use
with the legacy `FileTree` class. New projects should use
`preloadPathStoreFileTree` from `@pierre/trees/path-store`.
