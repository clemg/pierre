## Path-Store API

<Notice icon={<IconInfoFill />}>
  Import the path-store file tree from `@pierre/trees/path-store`.
</Notice>

The Path-Store API exposes `PathStoreFileTree`, an imperative class that manages
the full tree lifecycle: rendering, mutations, search, selection, drag and drop,
and cleanup. It works in any JavaScript environment — no framework required.

### Instance methods

| Method                           | Description                                                 |
| -------------------------------- | ----------------------------------------------------------- |
| `render({ containerWrapper })`   | Mount the tree into a parent DOM node                       |
| `hydrate({ fileTreeContainer })` | Hydrate a pre-rendered `file-tree-container` element        |
| `cleanUp()`                      | Unmount and release all resources                           |
| `add(path)`                      | Add a file or directory to the tree                         |
| `remove(path, options?)`         | Remove a file or directory (optionally recursive)           |
| `move(from, to, options?)`       | Move a file or directory to a new path                      |
| `batch(operations)`              | Apply multiple add/remove/move operations atomically        |
| `resetPaths(paths, options?)`    | Replace the entire file list                                |
| `getItem(path)`                  | Get a typed handle for the item at the given path           |
| `getSelectedPaths()`             | Return the currently selected paths                         |
| `getFileTreeContainer()`         | Return the root `file-tree-container` element               |
| `setGitStatus(entries?)`         | Update git status indicators                                |
| `setIcons(icons?)`               | Update the icon configuration                               |
| `openSearch(initialValue?)`      | Open the search input with an optional initial value        |
| `closeSearch()`                  | Close the search input                                      |
| `setSearch(value)`               | Set the search value programmatically                       |
| `isSearchOpen()`                 | Check if search is open                                     |
| `getSearchValue()`               | Get the current search string                               |
| `getSearchMatchingPaths()`       | Get paths matching the current search                       |
| `focusNextSearchMatch()`         | Focus the next search match                                 |
| `focusPreviousSearchMatch()`     | Focus the previous search match                             |
| `startRenaming(path?)`           | Start inline renaming for the given path (or focused item)  |
| `onMutation(type, handler)`      | Listen for mutation events. Returns an unsubscribe function |

### Legacy Vanilla JS API [toc-ignore]

The legacy `FileTree` class from `@pierre/trees` is still available. It uses
`initialFiles`, `FileTreeStateConfig`, and `setFiles` for updates. New projects
should prefer `PathStoreFileTree` from `@pierre/trees/path-store`.
